# Git Workflow

## Commit Policy

- Create small, focused commits.
- Use Conventional Commits for every commit message.

## Push Policy

Push when one of these occurs:

1. Completion of a functional development block.
2. Before starting a new development area.
3. Before ending a development session.
4. Before risky refactors or architectural changes.

## Versioning Policy

Versioning starts when the application has its first usable UI.

Format:

v0.1.0

Before that milestone, the project remains in pre-version development.
